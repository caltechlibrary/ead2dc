This site provides information about, and configuration for the Caltech Archives Open Archives Initiative (OAI) Data Provider (DP). Access to the configuration section requires an Archives staff login.

The DP supports the six OAI verbs and their arguments, with a few exceptions, as described below. The service is freely available. Any questions about this service should be directed to archives@caltech.edu

DP Base URL: https://apps.library.caltech.edu/ead2dc/oai

Example queries:

  • Identify
    • Identify
  • ListMetadataFormats
    • ListMetadataFormats
    • ListMetadataFormats&identifier=collections.archives.caltech.edu/repositories/2/archival_objects/106578
  • ListSets
    • ListSets
      • argument not supported: resumptionToken
  • ListIdentifiers
    • ListIdentifiers
    • ListIdentifiers&set=124
    • ListIdentifiers&set=124&from=2025-01-01&until=2026-12-31
    • ListIdentifiers&resumptionToken=124:2025-01-01:2026-12-31:250
      • arguments supported: set, from, until, resumptionToken
      • argument not supported: metadataPrefix
  • ListRecords
    • ListRecords
    • ListRecords&set=219
    • ListRecords&resumptionToken=219:000-00-00:999-99-99:250
      • arguments supported: set, from, until, resumptionToken
      • argument not supported: metadataPrefix
  • GetRecord
    • GetRecord&identifier=collections.archives.caltech.edu/repositories/2/archival_objects/106628
      • argument supported: identifier
      • argument not supported: metadataPrefix

Notes:

  • Metadata
    • Dublin Core is the only metadata standard supported at this time.
  • Flow control
    • ListIdentifiers and ListRecords return batches of 250 records, with a resumption token.
    • Resumption tokens are valid indefinitely. Note, however, that idempotency is not guaranteed as the underlying data may change at any time.
  • Sets
    • The set structure is flat, not hierarchical.
    • All records belong to one and only one set, so harvesting all sets will capture all records.

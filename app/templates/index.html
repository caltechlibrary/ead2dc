Encoded Archival Description to Dublin Core Conversion

The Caltech Archives use the Encoded Archival Description (EAD) XML output from ArchivesSpace to create a static XML repository, which in turn is used to provide OAI-PMH access to the digital collections. The static repository is inspired by the OAI Static Repository specification, but does not adhere to it strictly. The static repository contains Dublin Core (DC) records for digital resources found in the finding aid. Only records containing links to digital objects are included.

The OAI Data Provider adheres to the OAI standard and supports all the verbs (Identify, ListMetadataFormats, ListSets, ListIdentifiers, ListRecords, and GetRecord), resumption tokens, and sets. Only DC metadata is provided. Sets correspond to the archival collections in the Caltech Archives.

ead2dc - Main features and assumptions:

  • Both input and output are OAI-compliant XML files.
  • All 12 levels of EAD container are supported.
  • Titles are inherited down the container hierarchy.
  • All other metadata is mapped from the record containing the digital object references.
  • Records without digital object references are ignored (i.e. not mapped to the output file)

OAI Data Provider - Main features and assumptions:

  • The OAI Data Provider uses a static repository, i.e. it does not dynamically generate records.
  • DC metadata only.
  • Sets correspond to the archival collections and do not overlap.
  • Records are delivered in batches of 250.

Example queries

Base URL: https://apps.library.caltech.edu/ead2dc/oai

  • Identify
    • Identify
  • ListMetadataFormats
    • ListMetadataFormats
    • ListMetadataFormats&identifier=collections.archives.caltech.edu/repositories/2/archival_objects/106578
  • ListSets
    • argument not supported: resumptionToken
    • ListSets
  • ListIdentifiers
    • arguments supported: set, from, until, resumptionToken
    • argument not supported: metadataPrefix
    • ListIdentifiers&set=124&from=2023-01-01&until=2023-12-31
    • ListIdentifiers&resumptionToken=124:2023-01-01:2023-12-31:250
  • ListRecords
    • arguments supported: set, from, until, resumptionToken
    • argument not supported: metadataPrefix
    • ListRecords&set=219
    • ListRecords&resumptionToken=219:000-00-00:999-99-99:250
  • GetRecord
    • argument supported: identifier
    • argument not supported: metadataPrefix
    • GetRecord&identifier=collections.archives.caltech.edu/repositories/2/archival_objects/106628

Notes:

  • Metadata
    • Dublin Core is the only metadata standard supported at this time.
  • Flow control
    • ListIdentifiers and ListRecords return batches of 250 records, with a resumption token.
    • Resumption tokens are valid indefinitely. Note, however, that idempotency is not guaranteed as the underlying data may change at any time.
  • Sets
    • The set structure is flat, not hierarchical.
    • All records belong to one and only one set, so harvesting all sets will capture all records.
